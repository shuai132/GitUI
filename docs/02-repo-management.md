# 02. 多仓库管理

GitUI 支持多仓库同时挂载与快速切换。本中心模块负责仓库的生命周期管理、状态持久化以及多实例协作。

## 核心职责

- **名册管理**：维护已打开仓库的路径映射与唯一标识。
- **生命周期**：处理仓库的打开、克隆、初始化及关闭。
- **状态监控**：管理每个仓库的文件系统监听器，并协调防抖与过滤逻辑。
- **持久化**：确保用户打开的仓库列表及最后活跃仓库在重启后能正确恢复。

## 仓库操作流程

### 创建与克隆
- **统一入口**：通过前端 Store 协调创建逻辑，确保无论是 Clone 还是 Init，最终都收敛到统一的仓库注册流程。
- **后台执行**：长耗时的 Git 操作在后端异步执行，并通过事件流实时回传进度信息。
- **进阶处理**：Clone 过程中支持凭据回调映射、递归子模块以及浅克隆配置。

### 打开与激活
- **验证**：打开路径时由后端验证其 Git 有效性。
- **名册注册**：成功验证后分配唯一 ID 并启动对应的文件监控服务。
- **激活链**：仓库激活后，会自动触发前端各 Store（工作区、历史、分支、子模块等）的刷新逻辑。

## 状态持久化机制

系统区分两类持久化数据：
1. **名册数据**：存储在独立的配置文件中，包含已打开仓库的路径列表及活跃路径，由插件化存储引擎管理。
2. **UI 偏好**：存储在前端本地缓存中，包含侧栏宽度、面板高度、显示偏好等非核心业务数据。

## 切换与视图状态恢复

为保证无缝切换体验，系统实现了以下机制：
- **状态快照**：切换仓库时，当前仓库的视图状态（如选中提交、滚动位置、工作区缓存）会被记录。
- **多 Session 隔离**：如内置终端等模块，会为每个仓库维护独立的进程 Session，确保切换回来时状态保持。
- **延迟加载**：非活跃仓库仅保留基础路径信息，只有在切换激活时才发起完整的元数据加载。

## 文件系统监控 (Watcher)

- **全局监控**：监控范围覆盖整个工作目录以确保感知所有变动。
- **智能过滤**：后端在推送变动事件前，会根据仓库根目录的忽略规则（如 `.gitignore`）进行前置过滤，避免无效的 IPC 压力。
- **防抖逻辑**：多文件连续变动会被合并为单个更新事件，平衡实时性与性能。

## 多实例协作 (macOS)

- **独立进程**：支持在新进程中打开仓库，实现窗口间的完全隔离。
- **启动握手**：通过命令行参数与 Managed State 协调，确保新实例启动后能自动激活目标仓库。

## 界面定制

系统支持高度自定义的布局调整，包括侧栏宽度、列表高度等。所有拖动类操作均遵循“交互中响应、动作结束后持久化”的原则。
