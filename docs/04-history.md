# 04. 提交历史视图

历史视图是系统的核心，集成了提交图、日志列表、变更详情以及工作区入口。它旨在提供一个统一的界面来浏览版本演进并执行版本管理动作。

## 核心职责

- **版本溯源**：通过提交图与列表展示版本演进脉络，支持分页加载海量日志。
- **提交过滤**：支持基于文本、作者及哈希值的实时搜索与过滤。
- **关联展示**：展示分支标签、标签（Tags）以及远程同步状态。
- **详情透视**：提供提交元数据（作者、日期、完整信息）与文件差异的深度查看。

## 视图布局与交互

### 灵活布局
支持水平（左右）与垂直（上下）两种布局模式，满足不同屏幕尺寸下的查看习惯。所有面板大小与列宽均支持交互式调整并自动持久化。

### 虚拟滚动与分页
利用虚拟滚动技术确保在大仓库下的流畅滚动，并采用自动分页策略在用户接近底部时加载更多历史。

### 键盘导航
提供完整的键盘快捷键支持，允许用户在提交列表与文件清单间快速切换、移动选中项并触发常用动作。

## 数据集成逻辑

### 日志构建 (Revwalk)
后端采用启发式算法构造提交序列：
- **可达性分析**：区分已引用提交、丢失引用的提交（Unreachable）以及贮藏（Stash）。
- **结构化转换**：将 Git 内部的提交关系转换为前端易于渲染的平铺列表，并为提交图算法提供必要的拓扑信息。
- **辅助过滤**：在渲染时智能过滤掉 Git 内部的辅助提交（如 Stash 产生的索引/未追踪快照），保持视图整洁。

### 搜索与过滤
系统在前端实现实时的文本过滤，支持匹配标题、作者及 OID。在搜索模式下，系统会动态调整视图（如隐藏提交图）以优化信息密度。

### 标签与同步状态
- **分支/标签映射**：动态计算并展示指向特定提交的所有引用。
- **远程同步感知**：通过后台比对，为标签提供远程同步状态指示（如已同步、仅本地等）。

## 详情面板功能

### 元数据查看 (CommitInfo)
展示详尽的提交信息，支持长内容的横向滚动与文本的自由选中复制。文件变更列表支持按路径搜索并能关联到 Diff 查看器。

### 差异对比
集成三模态 Diff 查看器，详情见 `docs/06-diff-viewer.md`。

## 版本管理动作

通过上下文菜单提供丰富的管理能力：
- **分支管理**：基于特定提交创建分支或标签。
- **引用变更**：检出提交、重置分支（Soft/Mixed/Hard）。
- **内容迁移**：Cherry-pick、Revert。
- **信息修补**：支持对 HEAD 或历史提交进行消息修补（Reword）。

## 刷新策略

任何导致引用变动的操作（如切换分支、重置、Cherry-pick）都会触发自动的日志与分支状态刷新，确保 UI 与底层仓库状态实时同步。
