# 08. 远程操作

本模块负责与远程仓库的交互，涵盖 Fetch、Push、Pull 等核心同步命令，以及远程 URL 与凭据的管理。

## 核心职责

- **数据同步**：执行远程数据的拉取（Pull/Fetch）与推送（Push）。
- **凭据管理**：协调不同协议下的身份验证，包括 SSH 密钥对、HTTPS 凭据助手等。
- **冲突预防**：在执行非快进（Non-FF）推送或存在本地冲突时提供安全防护。
- **交互引导**：在大仓库、多远程节点场景下提供直观的节点选择与进度反馈。

## 网络协议与底层驱动

系统采用混合驱动策略以确保跨平台的高兼容性：

1. **HTTPS 协议**：主要通过内置的 Git 引擎库实现，集成系统级凭据管理器（如 Keychain、Credential Manager）。
2. **SSH 协议**：为了复用用户已有的 SSH 配置与代理环境，系统对于 SSH 类型的远端操作采用桥接宿主环境原生工具的策略。这确保了在不同操作系统下都能获得稳定、一致的验证体验。

## 同步流程设计

### Fetch 与状态更新
Fetch 操作不仅同步远程对象，还会原子化地触发本地分支、提交历史以及远程标签状态的刷新。系统会自动检测远程标签的同步情况并更新 UI 指示（详见 `docs/04-history.md`）。

### Pull 与合并策略
Pull 操作集成了 Fetch 与随后的合并。
- **模式支持**：默认支持快进合并（Fast-forward），并在允许的配置下支持产生合并提交。
- **安全性**：当检测到远程分支与本地存在偏离（Diverged）且无法直接合并时，系统会中止操作并提示用户手动介入。

### Push 操作
- **分支推送**：执行当前分支到对应上游分支的同步。系统遵循保守策略，不默认执行强制推送。
- **标签推送**：支持对特定标签进行远程同步，并提供防重写保护。

## 交互设计

- **多远程选择**：在配置了多个远程节点（Remotes）时，系统会提供交互式的选择菜单。
- **进度反馈**：网络同步过程中的进度（如接收对象、解析增量、检出文件）会实时回传并展示在 UI 界面上。
- **并发控制**：为每个仓库维护独立的同步忙碌状态，防止并发冲突。

## 凭据回馈机制

对于 HTTPS 协议，系统会尝试按顺序匹配多种凭据源（如系统代理、配置文件、SSH Agent 等），以最大限度地实现无感知认证。当所有尝试均失败时，会向用户反馈明确的认证失败信息。
