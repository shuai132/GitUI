# 06. Diff 查看器

Diff 查看器服务于 **WIP 工作区 diff** 和 **提交内 diff**。两种场景共享同一套后端数据契约和前端渲染组件。

## 核心职责

- **数据获取**：根据当前仓库路径、文件路径、是否已暂存或特定提交 OID，计算并返回结构化的差异数据。
- **多模式展示**：支持左右分栏（Side-by-Side）、行内（Inline）以及按块（By-Hunk）三种查看模式。
- **交互功能**：支持语法高亮、变更块快速跳转、图片/SVG 预览。

## 模块划分

- **后端 (`commands/diff.rs`)**：负责调用 Git 引擎获取差异。针对工作区 diff，会包含未追踪（Untracked）文件的内容。
- **前端组件 (`components/diff/`)**：
  - `DiffView`：主入口，管理模式切换、工具栏状态和公共交互逻辑。
  - `InlineDiff`：渲染行内流式布局，同时处理按块分组模式。
  - `SideBySideDiff`：渲染左右对比布局，处理行对齐。
  - `ImageDiff`：处理位图和 SVG 的特殊对比展示。
- **逻辑工具 (`lib/highlight.ts`, `lib/wordDiff.ts`)**：提供语法高亮映射和词级差异计算。

## 数据契约

后端返回的差异数据结构应包含以下关键信息（详情见 `src/types/git.ts::FileDiff`）：
- **文件路径**：旧路径与新路径（处理重命名）。
- **差异统计**：新增行数与删除行数。
- **Hunk 列表**：每个 Hunk 包含补丁头（Header）及具体的行数据。
- **Blob 引用**：旧版与新版的 Blob OID，用于按需拉取原始内容（如图片预览）。

## 关键特性实现策略

### 查看模式持久化
用户选择的查看模式会持久化到本地存储，以便在不同会话和仓库间保持一致。

### 语法高亮
前端根据文件扩展名映射到具体的语言类型，并调用高亮引擎进行按行渲染。为保证性能，高亮开关状态也是可持久化的。

### 变更跳转
通过收集当前视图中所有变更块的 DOM 位置，实现基于滚动位置的“上一个/下一个”精确跳转。

### 媒体预览
- **位图/SVG**：针对图片文件，不展示字符差异，而是直接拉取 Base64 字节流进行并排展示。
- **按需加载**：大文件预览受限于大小阈值，以避免内存和 IPC 通信过载。
- **WIP 兼容性**：对于工作区中尚未暂存的新文件，通过直接读取磁盘文件来获取最新内容。

## 性能取舍

- **回调节流**：在解析大型 Diff 时，后端应控制数据组装频率。
- **按需拉取**：除文本差异外，二进制内容的拉取是惰性的。
- **子集高亮**：仅注册主流编程语言的高亮支持，平衡首屏加载速度与功能覆盖面。
