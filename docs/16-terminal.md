# 16. 内置终端 (Terminal)

内置终端模块允许用户在不离开应用的情况下直接执行命令行操作，提供与仓库上下文紧密结合的交互式体验。

## 核心职责

- **会话管理**：为每个打开的仓库提供独立的 PTY（伪终端）会话。
- **状态保持**：支持跨仓库切换时的会话持久运行。
- **实时同步**：实现命令输入与控制台输出的高效双向同步。
- **UI 集成**：提供紧凑且符合整体设计风格的终端界面。

## 架构设计

### 后端 PTY 驱动
后端负责管理子进程的生命周期，通过高性能的桥接层捕获标准输出，并将其异步推送到前端。
- **多实例隔离**：每个仓库拥有独立的子进程空间。
- **流数据处理**：采用二进制编码传输，确保非 UTF-8 字符流及复杂 ANSI 转义序列的正确解析。

### 前端状态与渲染
前端负责维护会话名册，并为每个会话提供独立的渲染实例。
- **多标签体系**：支持在同一仓库下管理多个终端标签页。
- **持久化运行**：隐藏终端面板不会导致会话中止。任务（如长耗时构建）可以在后台继续执行。

## 关键设计决策

### 性能与渲染策略
- **惰性加载**：终端实例仅在首次变为可见时才执行初始化。
- **视图保护机制**：系统会监控容器的可见性，确保渲染引擎仅在有效的几何尺寸下运行，从而避免无效的重绘负载。
- **DOM 驻留**：非活动标签页在 DOM 中保留其状态，以实现秒级的瞬间切换。

### 交互体验
- **路径感知**：新启动的终端会话会自动将工作目录定位至对应仓库的根目录。
- **紧凑布局**：采用极简风格的标签栏设计，最大限度地增加命令行的可见区域。

## 数据流向

1. **输入流**：前端捕获键盘事件或粘贴数据，通过加密安全的通道发送至后端 PTY 的标准输入。
2. **输出流**：后端持续轮询子进程输出，并通过高效的推送机制告知前端渲染引擎。
3. **尺寸协调**：前端视图尺寸变动（如面板拖拽）会实时同步至后端，确保子进程能正确换行并排版。

## 未来扩展

- **硬件加速支持**：为超大吞吐量的输出场景引入更高效的渲染模式。
- **Shell 环境定制**：允许用户根据平台特性指定自定义的默认 Shell 程序。
