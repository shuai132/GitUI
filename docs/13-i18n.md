# 13. 国际化

GitUI 通过 `vue-i18n` v11 提供中英双语支持，默认跟随系统语言。语言选择在「设置 → 外观 → 界面语言」切换，立即生效，无需重启。

## 涉及模块

- `src/i18n/index.ts`：`createI18n` + `detectInitialLocale` + `applyAutoLocaleFromSystem` + `setLocale`，导出全局 `i18n` 实例和 `t` 函数
- `src/i18n/locales/zh-CN.ts` / `en.ts`：两个语言的全部翻译资源（按域组织在 `messages` 顶层）
- `src/stores/settings.ts`：`uiLanguage: 'auto' | 'zh-CN' | 'en'` 字段持久化，watch 触发 `applyLocaleFromSettings`
- `src/components/settings/AppearanceSection.vue`：3 卡片选择器（与主题、提交图等并列）
- `src/lib/errorMap.ts`：把 Rust 后端错误映射为 i18n key（`errors.*`）

## locale 解析

`uiLanguage` 用户档位：
- `'zh-CN'` / `'en'`：用户显式指定，直接生效
- `'auto'`：跟随系统（默认）

`auto` 模式下，初始 locale 按以下顺序**同步**确定（`detectInitialLocale`）：
1. localStorage `gitui.settings.v1` 的 `uiLanguage` 字段；若已是 `zh-CN | en` 则用之
2. localStorage `gitui.i18n.autoCache`（上次系统 locale 异步解析的结果缓存）
3. `navigator.language` 经 `normalizeLocale` 归一
4. 兜底 `'zh-CN'`

挂载后 store watch 在 `auto` 档异步调 `@tauri-apps/plugin-os` 的 `locale()`，归一后若与当前不同则切换并写回 `autoCache`，下次冷启动同步命中无闪烁。

`normalizeLocale`：`zh-*`（含繁体）→ `'zh-CN'`，其他 → `'en'`。要扩 `zh-TW` 等只改这一处。

## key 命名

```
<domain>.<section>.<item>
```

- 三级以内 camelCase
- 顶层域：`common / app / toolbar / sidebar / history / branches / branch / branchList / tag / submodule / workspace / diff / terminal / debug / settings / reflog / errorHistory / errors`
- 带参数用 vue-i18n 命名插值 `{name}`
- Git 术语和品牌名（branch / Pull / Push / Cherry pick / SF Mono / iTerm2 / Ghostty…）两种语言通常保持英文同字串
- 「自动 / 跟随系统」相关：复用 `settings.advanced.uiLanguageAuto`

## 添加新文案

1. 在 `src/i18n/locales/zh-CN.ts` 找对应域，加 key + 中文值
2. 在 `src/i18n/locales/en.ts` 镜像位置加同 key + 英文值
3. 组件里：`<script setup>` 加 `const { t } = useI18n()`，模板用 `{{ t('domain.key', { param }) }}`
4. 非组件（store / utils）：`import { i18n } from '@/i18n'` + `i18n.global.t('...')`

## errorMap 集成

`mapGitError(op, raw)` 返回 `FriendlyError { key, params?, fallbackText? }` 而不是字符串。Rust 后端目前仍发中/英混合错误，正则匹配后输出 `errors.<domain>.<reason>` key。`stores/errors.ts` 在 push 时通过 `i18n.global.t` 渲染为当前 locale 字符串存进 entry。

未命中任何 PATTERN 时兜底走 `errors.generic.unknown` + `{detail}` 原文，保证英文用户看到的是英文壳 + 原始细节。

约 20 条 key 覆盖 auth / remote / ref / push / pull / rebase / merge / worktree / cherrypick / config / repo / network 等场景，详见 `locales/*/errors`。

## 注意事项

- **品牌名不翻译**：SF Pro / Inter / iTerm2 / Ghostty 等在两个 locale 文件里值相同
- **Pinia store 使用 i18n**：导入 `i18n` 实例（不要用 `useI18n` composable，store 不在 Vue 组件上下文）
- **预设的 label 字段**：`UI_FONT_PRESETS` / `CODE_FONT_PRESETS` / `EXTERNAL_TERMINAL_PRESETS` 用 `labelKey: string`，UI 层 `t(preset.labelKey)` 渲染
- **Rust 后端错误**：暂不国际化，由 `errorMap.ts` 在前端做关键字识别再翻译
- **JS 注释**：保留中文注释，i18n 只覆盖用户可见文案
