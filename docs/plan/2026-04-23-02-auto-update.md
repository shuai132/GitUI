# 实现自动版本更新

## Context
为 GitUI 提供平滑的 OTA 应用自动升级体验。用户无需去 GitHub 手动下载新版本的安装包，应用可在后台或通过手动点击“检查更新”获取最新版本，下载后自动安装并重启。此功能依赖于 Tauri v2 的 `@tauri-apps/plugin-updater` 插件，以及由 `tauri-action` 结合 GitHub Releases 提供的更新源（`latest.json`）。

## 关键决策
- **更新源**：利用当前的 GitHub 托管仓库，配置 endpoint 为 `https://github.com/shuai132/GitUI/releases/latest/download/latest.json`（根据实际 repo owner 替换）。由于 `tauri-action` 会在触发 release 时自动生成 `latest.json`，此方案最轻量。
- **UI 触发点**：在“关于”面板里（`AboutInfo.vue`）添加一个检查更新的按钮，最符合用户习惯。
- **签名密钥**：通过 Tauri CLI 临时生成，之后交由用户配置到 GitHub Secrets，计划中只提供占位和环境变量。

## 验证方式
开发完成后：
1. 检查 `tauri.conf.json` 中的 `plugins.updater` 配置。
2. 运行 `npm run tauri dev`，打开“关于”页面点击“检查更新”，如果是最新版应弹出提示“已经是最新版”，如果有报错则检查网络或 endpoint 格式。
3. 提交代码并由用户配置 Secrets 之后触发一次 release 进行端到端验证。
