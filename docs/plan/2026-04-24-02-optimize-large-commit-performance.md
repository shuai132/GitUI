# 优化：大型提交详情加载性能

## 问题描述
当一个提交包含数千个文件变更或巨量代码行时，`get_commit_detail` 会因为一次性解析并传输所有文件的完整 Diff 内容而导致 UI 卡死约 1s 多。

## 解决方案
采用「摘要 + 异步分步 + 虚拟化列表」的极致优化策略：
1. **后端 (Rust)**: 
   - 增加 `get_commit_summary` 接口并支持 `include_stats` 开关。
   - 优化 `parse_diff_summary` 逻辑，仅在需要时统计增删行数。
2. **前端 (TS/Vue)**: 
   - **瞬间响应**: `selectCommit` 立即使用缓存数据弹出面板，通过 `setTimeout(0)` 将耗时请求推迟到下一帧。
   - **分步加载**: 先拉取文件列表摘要（瞬间），再异步补全增删统计数字。
   - **虚拟滚动**: 在 `CommitInfoPanel` 中引入 `@tanstack/vue-virtual`，即使文件数量过万，DOM 节点数依然保持恒定，彻底解决主线程阻塞。

## 效果
- 零延迟感：侧边栏在点击瞬间弹出。
- 极速加载：文件列表在 50ms 内呈现。
- 统计补全：增删数字在后台平滑更新，不干扰操作。
