# 2026-04-23-01-wip-tree-view

## Context
用户希望在 GitUI 的工作区（Stage 区域）支持以树状结构显示文件，并在文件列表（平铺）与树状结构之间提供切换功能。同时要求树状视图支持选中目录，并能对目录执行“历史”（File History / Blame）等相关上下文操作。

## 预期结果
1. 工作区 (WipPanel) 的 Header 区域新增一个切换按钮（Tree / List）。
2. 在 Tree 模式下，未暂存和已暂存文件会聚合成树状展示，文件夹支持折叠/展开。
3. 文件夹层级支持右键菜单，能够直接点击“History”查询该目录的变更历史。
4. 提供记忆功能（`localStorage` 保存展示偏好）。

## 关键决策
- **树状渲染方式**：由于大仓库可能有成百上千个改动，完全的 DOM 树会有性能问题。我们将产出的树**拍平成带有层级深度的线性列表**（包含展开/折叠状态），继续复用 `@tanstack/vue-virtual` 提升性能。
- **IPC与功能适配**：底层 git 命令对目录的 `git log -- <path>` 通常支持良好。需要验证并确保右键点击目录时，可操作其内容。

## 验证方式
1. 打开有较多深层级改动的仓库。
2. 切换至树状视图，确认能够正确折叠/展开，并保持流畅。
3. 选中某个文件夹，右键查看 History，确认弹出历史对话框并显示该目录相关的提交。
